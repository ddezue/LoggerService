# LoggerService

## Постановка задачи

При разработке приложений возникает проблема: нужно отслеживать ошибки и события, чтобы понимать, что происходит в программе. Если не вести логи, невозможно узнать, почему приложение упало или где возникла ошибка.

**Прикладная проблема:** в большом приложении разные части программы могут создавать свои логгеры. Это приведёт к тому, что логи разбегутся по разным файлам, а при одновременной записи возникнут конфликты.

**Решение:** создать службу логирования в единственном экземпляре, чтобы вся программа писала логи в одно место.

## Паттерн Singleton

**Какой паттерн:** Singleton (Одиночка)

**Цель применения:** гарантировать, что у класса `Logger` будет только один экземпляр во всём приложении.

**Какую задачу решает:**
- Предотвращает создание нескольких логгеров
- Исключает конфликты при записи в файл
- Даёт глобальную точку доступа к логгеру из любого места программы

## Реализация

- Приватный конструктор - запрещает создание через `new`
- Статическое свойство `Instance` - единственный способ получить логгер
- Методы `Info()`, `Warning()`, `Error()` - запись логов в консоль и файл
